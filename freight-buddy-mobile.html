Why a mobile version of the same app?

As someone who is not very creative, I wanted to focus on picking up a new technology by myself - learning how to create a mobile app without prior experience, and not having to spend too much time on ideation due to time constraints.

That said, I did not expect an unforseen networking issue and could not implement all the features I had planned. The gruelling process of googling for answers taught me more about networking as well. More on than in the learning points section below.

Description

Freight Buddy was created to help people (requesters) gain access to their beloved overseas products by enlisting the help of others overseas to purchase the products and ship them over to the requesters.

Motivation

Ever once in a while, there will be a product sold overseas that my friends and I want but can't be found on any e-commerce platform, or the shop that sells the product does not ship to my country. Thus sprouted the idea of a peer-to-peer platform which could solve this problem, where people finally have an avenue for getting their hands on the product they dearly want.

Main Learning Points

  • Implemented a React Native app by self-learning through reading documentation and googling.
  • Implemented a workaround for a bug which prevented connection between the local repository (server) and mobile devices that are on the same WIFI network. Workaround was to install ngrok package to provide the local repository with an internet IP address so any device could connect to the local repository through the internet.
